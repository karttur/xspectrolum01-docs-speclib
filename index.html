A brewing project

The xspectrolum spectral system is a project for acquiring and analysing spectral data, developed with xSpecre's xSpectrometer+ in mind. When writing this, 14 january 2024, the system conists of 4 different projects on GitHub:
  1. Soil spectral libraries a public online set of turorials with open data for analysing soil spectra using Machine Learning,
  2. xspectrolum_v01rB a private repo for the firmware of the xSpectrometer+
  3. xspectrolum01p-mojjang_pydevÂ a private repo with the python code that accessesing the xSpectrometer+ through a usb interface, and
  4. xspectrolum01superproj a privare superproject repo with 20 provate submodules (repos) for managing spectral campaigns.

Soil spectral libraries

This stanad alone blog is a short hands-on training for working with the Open Soil Spectral Library (OSSL) and other spectral data. The intention is to integrate data from the xSpectrometer+ in the library and thus model soil properties based on spectra from the xSpectrometer+.

xspectrolum_v01rB

The repo xspectrolum_v01rB is the development site for the firnmware for version 0.81 of the xSpectrometer+. It is a private repo.

xspectrolum01p-mojjang_pydev

The repo xspectrolum01p-mojjang_pydev is for development of the python code that sends and receive information to the xspectrolum_v01rB firmware. It is a private repo.

xspectrolum01superproj

The superproject repo xspectrolum01superproj is an umbrella project/repo for 20 submodules/repos (each constituting a Python package) that is intended to be the starting point for buidling the user system for managing, capturing and analyzing spectral data using the xSpectrometer+. Both the superproject and all the submodules are private repos. The post Spectral system setup: 3. Super project structure in this blog lists all the submodules belonging to the superproject.
